## Project Overview

**智雨天气 (RainWeather)** - A modern Flutter weather forecasting application with intelligent location services and AI-enhanced features.

- **SDK**: Flutter 3.9.2+, Dart 3.9.2+
- **State Management**: Provider pattern
- **API**: weatherol.cn for weather data
- **Database**: SQLite for local caching

## Architecture

### Directory Structure
```
lib/
├── constants/          # App constants, colors, themes, MD3 design tokens
├── models/             # Data models with JSON serialization
├── providers/          # State management (WeatherProvider, LocationProvider, CitiesProvider, AIInsightsProvider, RefreshCoordinator, ThemeProvider)
├── screens/            # Main pages and feature screens
├── services/           # Business logic, API, location, database
├── utils/              # Helper utilities (logging, error handling, etc.)
└── widgets/            # Reusable UI components
```

### State Management (New Architecture - Facade Pattern)

**核心架构**：使用Facade（外观）模式，WeatherProvider作为统一门面，内部委托给专门的子Provider

```
WeatherProvider (Facade - 统一入口)
    ├─→ LocationProvider (定位管理)
    ├─→ CitiesProvider (城市管理)
    ├─→ AIInsightsProvider (AI摘要)
    └─→ RefreshCoordinator (定时刷新)
         ↓
    Services Layer (WeatherService, LocationService, DatabaseService等)
```

**Provider职责划分**：

- **WeatherProvider** (Facade - 统一入口)
  - 作为WeatherProvider的统一门面，对外保持向后兼容的API
  - 内部委托给子Provider处理具体逻辑
  - 管理天气数据和缓存
  - 协调Provider间的数据流转

- **LocationProvider** (定位管理)
  - GPS定位管理（当前定位、原始定位）
  - 定位防抖（5分钟内不重复定位）
  - 位置切换（GPS定位 ↔ 城市天气）
  - 提供定位刷新接口

- **CitiesProvider** (城市管理)
  - 管理主要城市列表（增删改查、排序）
  - 管理城市天气数据
  - 城市搜索功能
  - 城市天气刷新状态管理

- **AIInsightsProvider** (AI摘要)
  - 生成每日天气摘要
  - 生成15天天气趋势摘要
  - 生成通勤建议
  - AI生成状态管理

- **RefreshCoordinator** (定时刷新)
  - 管理1小时定时刷新
  - 应用前后台状态管理
  - Timer生命周期管理

- **ThemeProvider** (主题管理) - 保持不变
  - 管理主题模式（light/dark/system）
  - 管理颜色方案

**数据流向**：

```
LocationProvider (定位) → WeatherProvider (数据聚合) → UI (显示)
                         ↓
                    AIInsightsProvider (AI摘要)
                         ↓
                    CitiesProvider (城市管理)
                         ↓
                    RefreshCoordinator (定时刷新)
```

**关键改进**：

1. **职责分离**：每个Provider专注一个领域，单一职责原则
2. **向后兼容**：WeatherProvider的对外API保持不变，UI无需修改
3. **易于测试**：每个Provider可独立测试
4. **可维护性**：代码更清晰，降低复杂度

### Location Services (Multi-Provider Fallback Chain)
The app uses a sophisticated location system with multiple providers and automatic fallback:

1. **Primary**: GPS/Geolocator (best accuracy)
2. **Fallback 1**: Amap (高德) - for China region
3. **Fallback 2**: Tencent Location (腾讯) - alternative China provider
4. **Fallback 3**: IP-based location - last resort

**Key variables** in WeatherProvider:
- `_currentLocationWeather`: Weather data for current GPS location (never changes until new location)
- `_originalLocation`: Original location info from GPS (never changes)
- `_currentWeather`: Currently displayed weather (may be city weather)
- `_currentLocation`: Currently displayed location (may be city location)
- `_isShowingCityWeather`: Boolean flag indicating if showing city data vs location data

### Data Flow
```
LocationService → WeatherProvider → WeatherService → API
                  ↓
            Database Cache ← ← ← ← ← ← ←
                  ↓
                UI Updates
```

## Important Concepts

### Three Types of Weather Alerts (Do Not Confuse)

1. **天气提醒 (Smart Alerts)** - AI-analyzed weather alerts
   - Generated by `WeatherAlertService` based on weather data analysis
   - Only for current location, not for saved cities
   - Shown on TodayScreen

2. **气象预警 (Official Alerts)** - Raw API alerts from weather service
   - Direct from `weather.current.alerts` API response
   - Shown as icon on city cards and city weather screens
   - Available for all cities

3. **通勤提醒 (Commute Advice)** - Commute time suggestions
   - Generated by `CommuteAdviceService`
   - Only for current location during user-defined commute hours
   - Shown on TodayScreen with expandable card

### Theme System
- Uses Material Design 3 with ThemeExtension
- Three theme modes: light, dark, system
- Multiple color schemes: blue (default), green, purple, orange
- All colors accessed via `AppColors` class (call `AppColors.setThemeProvider()` before use)

### Widget Data Rule
**Home widgets MUST always display current location weather**, not the city weather user is currently viewing. Use `weatherProvider.currentLocationWeather` and `weatherProvider.originalLocation` for widget updates.

## Key Services

### State Management Providers
- **WeatherProvider** (Facade): 统一入口，协调子Provider，管理天气数据
- **LocationProvider**: GPS定位管理，定位防抖，位置切换
- **CitiesProvider**: 城市列表管理，城市搜索，城市天气数据
- **AIInsightsProvider**: AI摘要生成，通勤建议生成
- **RefreshCoordinator**: 定时刷新协调器，应用前后台管理
- **ThemeProvider**: 主题模式管理，颜色方案

### Business Logic Services
- **WeatherService**: Fetches weather data from API
- **LocationService**: Manages GPS location with fallback providers
- **DatabaseService**: SQLite operations for caching
- **AIService**: AI content generation (summaries, advice)
- **WeatherWidgetService**: Android home widget updates
- **NotificationService**: Local push notifications
- **CommuteAdviceService**: Commute time weather advice
- **WeatherAlertService**: Smart weather alerts analysis

## Data Models

### WeatherModel Structure
- `current.current`: Current weather conditions
- `current.air`: Air quality data
- `current.alerts`: Official weather alerts (气象预警)
- `forecast24h`: 24-hour forecast (24 items)
- `forecast15d`: 15-day forecast with AM/PM weather
- `sunMoonData`: Sunrise/sunset and moon phase

### LocationModel Key Fields
- `district`: Most important field for city matching and display
- `city`: City name for API queries
- `province`: Province for precision
- `lat/lng`: Coordinates
- `adcode`: Administrative code for city data matching

## Important Design Patterns

### Material Design 3 Cards
- Card radius: 8px
- Card spacing: 12px vertical
- Screen padding: 12px horizontal
- Card elevation: 2-4
- Inner small cards: 4px radius, 0.15/0.25 opacity backgrounds

### Color Constraints
- **Avoid blue colors in inner small cards** - poor contrast with dark blue app background
- Recommended: Orange (#FFB74D), Green (#64DD17), Red, Amber
- Use `AppColors` constants for theme-adaptive colors

### AI Content Display
Use `AIContentWidget` for progressive loading:
1. Show skeleton immediately
2. Fetch AI content
3. Fade in content with animation
4. Fallback to default content on error
